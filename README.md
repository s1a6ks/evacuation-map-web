## 🌟 Про Проект

Проект розроблено як сучасний інструмент для автоматизації рутинної роботи інженерів, архітекторів та інспекторів пожежної безпеки. замість ручного малювання стрілок, система дозволяє створити цифрову модель будівлі з усіма її характеристиками.

Тут ми поєднали **HTML5 Canvas** для малювання 2D-планів, **Теорію графів** для забезпечення оптимізації маршрутів та **ДБН В.1.1-7** для автоматичного виявлення небезпечних (тупикових) зон.

> [!NOTE]  
> Проект розроблено в рамках підготовки Бакалаврської кваліфікаційної роботи.

## 🔥 Ключові Можливості

### 🎨 1. Інтерактивний 2D Редактор (Високопродуктивний Canvas)
*   **Векторне малювання**: Інтуїтивне створення стін, дверей, сходів та вікон.
*   **Smart-сітка**: Вирівнювання (Grid Snapping) для точних масштабних планів.
*   **Авто-детекція кімнат**: Рендер-двигун сам розпізнає замкнуті простори та прораховує квадратні метри.

### 🧠 2. Smart-Алгоритми (Графовий двигун)
*   **Математичне покриття**: Конвертування малюнку в граф (вузли - приміщення, грані - двері/проходи).
*   **Dijkstra's Pathfinding**: Прорахунок та анімація найшвидших та найбезпечніших шляхів до виходів.
*   **Таймінг евакуації**: Оціночний час врятування на основі дистанцій та пропускних здатностей дверей.

### 🛡️ 3. Пожежна Експертиза (по ДБН)
*   🔴 **Детекція тупиків**: Авто-маркування кімнат, що не мають безпечної альтернативи виходу.
*   ⚠️ **Метрика дистанцій**: Аналіз відповідності довжини шляху до норм (< 25 метрів).
*   🟠 **Аналіз "By-pass"**: Візуалізація зон потенційного утворення натовпу ("вузькі горлишка").

## 🚀 Встановлення та Запуск

Ви можете легко розгорнути проект локально у два кроки.

### 1. Frontend (Клієнтська частина)
Перейдіть до директорії `evacuation-frontend` та встановіть залежності:
```bash
cd evacuation-frontend
npm install
npm run dev
```
> [!TIP]
> Додаток буде доступний за адресою `http://localhost:5173`.

### 2. Backend (Серверна частина)
Перейдіть до директорії API та запустіть .NET середовище:
```bash
cd EvacuationSystem.Api/EvacuationSystem.Api
dotnet build
dotnet run
```
> Сервер підніметься (зазвичай портом 5000/5001), база SQLite створиться автоматично (code-first міграції).
