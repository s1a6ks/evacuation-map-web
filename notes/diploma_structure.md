#### 1.3 Теоретичні основи графового моделювання
- Формальне визначення графа G = (V, E, w)

- Зв'язність графа як критерій безпеки евакуації

#### 2.2 Алгоритми пошуку евакуаційних маршрутів

##### 2.2.1 Алгоритм Дейкстри
- Математичний опис
- **Формула (2.1):** релаксація — d[v] = min(d[v], d[u] + w(u,v))
- Реалізація з пріоритетною чергою (бінарна купа)
- Складність: O((|V| + |E|) · log|V|)
- Особливість: досліджує всі напрямки рівномірно → гарантія оптимальності

##### 2.2.2 Алгоритм A*
- **Формула (2.2):** оцінна функція f(n) = g(n) + h(n)
  - g(n) — фактична відстань від старту
  - h(n) — евристична оцінка до цілі
- **Формула (2.3):** евристика — h(n) = min(√((xₙ-xₑ)² + (yₙ-yₑ)²)) для всіх виходів e
- Допустимість евристики: h(n) ≤ d*(n, goal)
- Складність: залежить від якості евристики, у кращому випадку значно менше Dijkstra

##### 2.2.3 Порівняльний аналіз алгоритмів
- **Таблиця 2.1** — Порівняння A* та Dijkstra:
  - Час (мс), переглянуті вузли, оптимальність, спрямованість пошуку
- Обґрунтування включення обох: Dijkstra для повного аналізу, A* для швидкого пошуку

#### 2.3 Автоматична побудова графа з плану будівлі
- **Алгоритм Flood Fill:**
  - Сканування canvas-сітки (150×100 клітинок)
  - Маркування порожніх областей (не стіни)
  - Об'єднання клітинок у кімнати
  - **Рис. 2.2** — Етапи: Canvas → Flood Fill → Кімнати → Граф
- **Генерація вузлів:**
  - По одному на кімнату (центроїд) + по вузлу на двері/виходи/сходи
- **Генерація ребер:**
  - З'єднання кімнат через двері (вага = евклідова відстань)
- **Формула (2.4):** обчислення ваги ребра w(u,v) = √((x₁-x₂)² + (y₁-y₂)²) × scale

#### 2.4 Багатоповерхова та мульти-кімнатна маршрутизація
- **Зв'язування сходів:** stairLinks: { "floorId:x:y": { toFloorId, toX, toY } }
- **Алгоритм findMultiFloorRouteWithLinks():** 
  - Пошук маршруту до сходів → перехід на інший поверх → пошук до виходу
  - Результат: [{ floorId, path }, ...]
- **Мульти-кімнатний режим:**
  - Вибір кількох кімнат → побудова маршрутів для кожної
  - Кольорова диференціація (8 кольорів)
- **Режим загального плану:** маршрути від усіх кімнат
- **Блокування елементів:** фільтрація заблокованих дверей/виходів для сценарного моделювання
- **Рис. 2.3** — Схема мульти-кімнатної евакуації

#### 2.5 Модель аналізу відповідності ДБН В.1.1-7
- 8 критеріїв перевірки (реалізовано в `evacAnalysis.js`):
  1. Площа поверху → мінімум виходів (≤300м²=1, 300–1000=2, >1000=3)
  2. Найдальша точка від виходу (4 кути кожної кімнати, ≤25м)
  3. Відстань від центроїдів кімнат до виходів
  4. Тупикові кімнати (вузли з degree ≤ 1)
  5. Зв'язність графа (всі кімнати досяжні від виходу)
  6. Вузькі місця (bottlenecks — вузли з degree ≥ 3)
  7. Кількість дверей на маршруті
  8. Час евакуації (відстань / 1.4 м/с)
- Автоматична генерація рекомендацій (ok / warning / error)
